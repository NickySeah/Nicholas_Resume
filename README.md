# Nicholas Seah - Software Engineering Resume

[![Build LaTeX Resume](https://github.com/NickySeah/Nicholas_Resume/actions/workflows/build-resume.yml/badge.svg?branch=master)](https://github.com/NickySeah/Nicholas_Resume/actions/workflows/build-resume.yml)

This repository contains my ATS-friendly software engineering resume written in LaTeX.

It also includes a GitHub Actions CI/CD workflow that automatically compiles the latest PDF on every push to the `master` branch and uploads it as a downloadable artifact.

## 🚀 Latest Release (Download)

To download the newest compiled resume PDF:

1. Open the repository on GitHub.
2. Click the **Actions** tab.
3. Open the latest successful run of **Build LaTeX Resume**.
4. Scroll to the **Artifacts** section.
5. Download the artifact named **Nicholas_Seah_Resume**.
6. Unzip it to get `main.pdf`.

This is the most up-to-date compiled version generated by CI.

## 🛠️ Local Development Setup

### Prerequisites

- **VS Code**
- **LaTeX Workshop** extension for VS Code
- A LaTeX distribution:
  - **MiKTeX** (Windows), or
  - **MacTeX** (macOS)
- **Strawberry Perl** (Windows) for `latexmk`

### Steps

1. Clone the repository:

```bash
git clone <your-repo-url>
cd NicholasResume
```

1. Compile the resume using `latexmk`:

```bash
latexmk -pdf -interaction=nonstopmode -halt-on-error main.tex
```

1. Open the generated PDF:

- Output file: `main.pdf`

### Optional cleanup

```bash
latexmk -c
```

## ⚙️ CI/CD Pipeline

The workflow file is located at `.github/workflows/build-resume.yml`.

On each push to `master`, GitHub Actions performs the following:

- Checks out the repository using `actions/checkout@v4`
- Compiles `main.tex` on `ubuntu-latest` using `xu-cheng/latex-action@v3`
- Uploads `main.pdf` as an artifact using `actions/upload-artifact@v4`
- Artifact name: `Nicholas_Seah_Resume`

This ensures every code update has a reproducible, downloadable PDF build.
